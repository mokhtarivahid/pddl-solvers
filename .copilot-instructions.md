## Core Guidelines

### Text-Only Communication
- **No emoji or special characters**: Never use characters like ✅, 🎯, ❌, 🚀, 📁, 📝, etc. in code, comments, docstrings, print statements, or documentation
- **Use plain text**: Communicate status, emphasis, and information using clear descriptive text only
- **Professional tone**: Keep all code communication simple and professional

# GOOD - Plain text

### Keep It Simple Principle
- **Minimal solutions**: Choose the simplest approach that works effectively
- **Avoid over-engineering**: Don't add complexity for features that might be needed later
- **Focus on functionality**: Prioritize working code over elaborate implementations
- **Clean codebase**: Remove unnecessary files, comments, and code regularly

### File Organization and Cleanup
- **Proper file placement**: Use appropriate directories (.github/ for project metadata, tests/ for testing)
- **Remove temporary files**: Don't leave demo files, test outputs, or debugging scripts in the codebase
- **Maintain .gitignore**: Keep generated files and build artifacts out of version control
- **Organized structure**: Follow the established project structure and naming conventions

### Documentation Standards
- **Purposeful documentation**: Write documentation that serves a clear function, not decorative text
- **Keep it current**: Update documentation when code changes, remove outdated information
- **Simple explanations**: Use plain language without excessive formatting or decoration
- **Practical examples**: Include working code examples rather than theoretical descriptions
